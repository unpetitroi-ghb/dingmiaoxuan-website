# AI 魔法绘本 · 变现说明（面向中国市场）

当前已实现的变现逻辑与后续可扩展方式，便于你在不增加大额投入的前提下尽快产生收入。

---

## 一、已实现逻辑

### 1. 免费额度

- **规则**：每个访客（按浏览器 cookie `guest_id` 区分）可**免费创作 1 本**完整绘本。
- **触发点**：用户完成「选角」后，在第二步点击「分析线索并推荐故事方向」时，会创建一条故事记录并占用额度；第二次再创作时，接口返回 **402**，前端弹出付费墙。
- **环境变量**：`FREE_STORY_LIMIT=1`（默认 1，可按需改为 0 或 2 做活动）。

### 2. 付费墙与定价页

- **付费墙**：当接口返回 402 时，创作页会弹出「继续创作需解锁」弹窗，展示：
  - 单本解锁 **9.9 元/本**
  - 月度会员 **19.9 元/月** 无限本
  - 支付方式由下方环境变量控制。
- **定价页**：`/pricing`，用于对外说明价格与支付方式，并可作为推广落地页。

### 3. 支付方式配置（零成本起步）

在 `.env` 或云环境变量中配置其一或组合使用：

| 变量 | 说明 | 推荐用途 |
|------|------|----------|
| `NEXT_PUBLIC_PAYMENT_LINK` | 支付/打赏页面链接 | **爱发电**、**面包多**、**知识星球** 等创作者平台的「赞助/购买」链接，用户跳转后付款，你手动或通过兑换码开通 |
| `NEXT_PUBLIC_WECHAT_REWARD_IMAGE` | 微信赞赏码图片 URL | 将赞赏码上传到图床（如 GCS 桶、又拍云），把公网 URL 填在此处，付费墙和定价页会展示「扫码赞赏，联系客服开通」 |

**零成本起步建议**：

1. 注册 **爱发电**（afdian.net）或 **面包多**（mianbao.io），创建「单次赞助 9.9 元」「月度 19.9 元」等方案，把对应链接设为 `NEXT_PUBLIC_PAYMENT_LINK`。
2. 在微信生成「赞赏码」，截图上传到现有 GCS 桶或任意图床，把图片 URL 设为 `NEXT_PUBLIC_WECHAT_REWARD_IMAGE`。
3. 用户付款后，通过 **手动在数据库里给该访客放行**（见下）或后续做「兑换码」功能。

### 4. 手动开通（当前无支付接口时）

用户通过爱发电/赞赏付款后，你可手动为其「放行」一次或多次：

- **方式 A**：在数据库中把该用户的某条 `Story` 的 `guest_id` 改成一个已用尽的旧 guest_id，相当于把这次创作从「新用户」名下挪走，该新用户下次仍会按新 guest 再享 1 次免费（不推荐，仅作理解用）。
- **方式 B（推荐）**：在数据库里**新增一张表**记录「已付费 guest_id 或兑换码」，后端在 `analyze-clues` 里除检查 `Story` 条数外，再查「该 guest 是否已购买单本/会员」，若已购买则不再拦截。下面「二、扩展方向」里会写简单表结构建议。

## 二、扩展方向（有收入后再做）

1. **兑换码**：在后台生成「单本券」「月会员券」，用户付款后你发兑换码，用户在站内输入兑换码后，将该 guest_id 与权益写入数据库，`analyze-clues` 校验通过即可继续创作。
2. **微信登录**：接入微信开放平台「网站应用」或「公众号内 H5」，用 unionid/openid 作为用户 ID，替代 guest_id，可做「每用户 1 次免费」或「会员」逻辑。
3. **微信支付 / 支付宝**：有营业执照和商户号后，接入官方支付（或通过 虎皮椒、易支付 等聚合），在付费墙内直接调起支付，支付回调后自动开通，无需人工。
4. **降低首单门槛**：例如「分享到朋友圈可再得 1 次免费」或「邀请 1 位好友各得 1 次」，用分享/邀请接口或落地页参数区分来源，在 quota 逻辑里增加一次免费机会。

## 三、成本与定价参考

- 单本绘本成本大致来自：**视觉分析（混元/百度/本地） + DeepSeek 脚本 + 即梦多页出图**。按当前调用量估算单本约 1～3 元级别（视页数与模型定价而定）。
- **9.9 元/本**、**19.9 元/月** 在亲子类产品里较常见，便于冲动消费；若你观察到转化率不错，可再试 14.9/本 或 29.9/月 做 A/B。

## 四、数据库与部署

- 已为 `Story` 表增加 `guest_id` 字段。**首次部署或拉代码后**在项目根目录执行一次：
  ```bash
  npx prisma generate
  npx prisma db push
  ```
  （若使用迁移流程则：`npx prisma migrate dev` 本地生成迁移，生产环境 `npx prisma migrate deploy`。）
- 付费墙和定价页会读取 `NEXT_PUBLIC_PAYMENT_LINK`、`NEXT_PUBLIC_WECHAT_REWARD_IMAGE`，在 `.env` 或云环境变量中配置后，需**重新构建**前端（`npm run build`）才会生效。

总结：当前已具备「免费 10 本 + 超量付费墙 + 定价页 + 爱发电/赞赏码」的完整路径；先接上爱发电或赞赏码尽快收单，再根据收入情况做兑换码或微信支付，可以控制投入、优先验证愿意付费的用户规模。管理后台见 **docs/ADMIN.md**。
