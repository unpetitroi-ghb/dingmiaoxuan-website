# 腾讯云 Ubuntu 部署指南

目标：把代码、PostgreSQL、视觉服务都放到腾讯云 `152.136.155.145`，用 Cursor Remote-SSH 已能连到 `ubuntu@152.136.155.145`。**推荐：代码先推到 GitHub，再在腾讯云上从 GitHub 克隆。**

---

## 零、先把代码推到 GitHub（在本地电脑做一次）

在**你本地**项目目录执行（已配置过 Git 且仓库在 GitHub 可跳过）：

```bash
cd /path/to/dingmiaoxuan
git status   # 确认 .env、gcs-key.json 没被跟踪（已在 .gitignore）
git add .
git commit -m "准备部署到腾讯云"
git remote add origin https://github.com/你的用户名/你的仓库名.git
git push -u origin main
```

- 若仓库已存在且是 `main` 以外的分支，把上面 `main` 改成你的分支名。
- `.env*` 和 `gcs-key.json` 已在 `.gitignore` 里，**不会**被推到 GitHub，密钥安全。
- 之后改代码：本地 `git push`，在腾讯云上 `git pull` 再 `docker compose up -d --build` 即可更新。

## 二、服务器准备（在腾讯云 Ubuntu 上做一次）

用 Cursor 的 **Remote - SSH** 连到 `ubuntu@152.136.155.145`，在终端执行。

### 1. 安装 Docker 与 Docker Compose

```bash
sudo apt-get update
sudo apt-get install -y ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a644 /etc/apt/keyrings/docker.asc
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo "$VERSION_CODENAME") stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
sudo usermod -aG docker ubuntu
```

登出再登录（或新开一个 SSH 会话），使 `docker` 组生效。

### 2. 从 GitHub 克隆代码

在腾讯云服务器上（Cursor Remote-SSH 连到 `ubuntu@152.136.155.145` 后的终端）：

```bash
cd ~
git clone https://github.com/你的用户名/你的仓库名.git dingmiaoxuan
cd dingmiaoxuan
```

- 若仓库是私有的，需在服务器上配置 GitHub 访问（二选一）：  
  - **HTTPS**：`git config --global credential.helper store`，首次 `git clone` 时输入 GitHub 用户名 + Personal Access Token（不要用密码）；  
  - **SSH**：在服务器执行 `ssh-keygen -t ed25519 -C "ubuntu@tencent"`，把 `~/.ssh/id_ed25519.pub` 内容加到 GitHub → Settings → SSH and GPG keys，然后用 `git clone git@github.com:你的用户名/你的仓库名.git dingmiaoxuan`。
- 以后更新代码：`cd ~/dingmiaoxuan && git pull`，再按需 `docker compose up -d --build`。

## 三、环境变量与密钥

在服务器上 `~/dingmiaoxuan` 目录创建 `.env`（不要提交到 Git）：

```bash
cd ~/dingmiaoxuan
nano .env
```

填入（按实际修改）：

```env
# 数据库（与 docker-compose 里 db 一致）
POSTGRES_USER=dingmiaoxuan
POSTGRES_PASSWORD=你的强密码
POSTGRES_DB=dingmiaoxuan

# 外部 API
DEEPSEEK_API_KEY=sk-xxx
JIMENG_API_KEY=AKLTxxx
JIMENG_SECRET_KEY=xxx

# GCS（图床）
GCS_BUCKET_NAME=你的桶名
# 可选：视觉模型，默认已是 openai/clip-vit-base-patch32
# VISION_MODEL=openai/clip-vit-base-patch32

# 网站对外地址（用于分享链接等）
NEXT_PUBLIC_BASE_URL=https://你的域名
# 若暂时没域名，可先写 http://152.136.155.145:3000
```

把本地的 **GCS 密钥 JSON** 上传到服务器同名路径，例如：

```bash
# 在本地执行，把 gcs-key.json 传到服务器
scp /path/to/your/gcs-key.json ubuntu@152.136.155.145:~/dingmiaoxuan/gcs-key.json
```

## 四、构建并启动

在服务器上（已 `cd ~/dingmiaoxuan`）：

```bash
docker compose up -d --build
```

首次构建会较久（含 vision 的 PyTorch/CLIP）。查看状态：

```bash
docker compose ps
docker compose logs -f
```

等 `vision` 健康通过后，app 才会起来。**首次启动后需要执行一次数据库迁移**：

```bash
docker compose exec app npx prisma db push
```

## 五、访问与反向代理（可选）

- **直接访问**：`http://152.136.155.145:3000`（确保腾讯云安全组放通 3000、5001、5432 若需外网访问）。
- **用域名 + HTTPS**：在同一台机上装 Nginx/Caddy，反代到 `http://127.0.0.1:3000`，并配置 SSL（如 Let’s Encrypt）。把 `.env` 里 `NEXT_PUBLIC_BASE_URL` 改为 `https://你的域名`。

## 六、常用命令

```bash
cd ~/dingmiaoxuan
docker compose ps          # 状态
docker compose logs -f app # 应用日志
docker compose logs -f vision
docker compose restart app
docker compose down        # 停掉并删除容器（数据在 volume postgres_data 里）
```

数据持久化：PostgreSQL 数据在 Docker volume `postgres_data`，`docker compose down` 不会删；若需清空数据库可 `docker compose down -v`（慎用）。
